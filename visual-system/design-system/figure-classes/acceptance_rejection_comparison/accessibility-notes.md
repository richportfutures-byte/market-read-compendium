# Acceptance Rejection Comparison Accessibility Notes

These notes are candidate design intent until verified on an actual export.

## Redundant Encoding

- Accepted or defended outcome uses hold shape, value block, and direct label in addition to accepted color.
- Rejected, trapped, or covering-only outcome uses failure path, forced-exit markers, exhaustion marks, and direct label.
- No-trade uses neutral watch state, incomplete-evidence icon, and "no classification earned" label.
- Shared setup is repeated by position and connector lines, not color alone.

## Grayscale Survival

The comparison must survive grayscale:

- accepted/defended branch has value-building block and shallow pullback hold;
- trapped branch has reentry and forced-exit path;
- covering-only has loss of reclaim and exhaustion;
- demand-confirmed has pullback hold and renewed initiative;
- no-trade has neutral decision strip.

## Contrast

Essential labels and marks use verified high-contrast token roles. Muted text is limited to region hints and metadata. Price and numeric labels use the monospace family.

## Motion And Static Equivalent

Interactive toggle specs still require a complete static export. The static plate must show the shared setup and all outcome states needed for the read.

## Alt Text Requirements

Published figures need authored alt text naming:

- the shared setup;
- the post-event evidence test;
- each outcome branch;
- the professional read and decision consequence;
- the forbidden shortcut the figure prevents.
