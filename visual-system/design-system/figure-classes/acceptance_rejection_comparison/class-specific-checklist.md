# Acceptance Rejection Comparison Class-Specific Checklist

This checklist blocks failures. It does not approve any figure.

## Conceptual Truth

- [ ] BLOCK if the shared setup is not visible before outcomes.
- [ ] BLOCK if classification happens at the break, reclaim, or burst.
- [ ] BLOCK if deciding evidence is not post-event behavior.
- [ ] BLOCK if a defended breakout is labeled or implied as a trap.
- [ ] BLOCK if covering is labeled fresh demand before first-pullback evidence.
- [ ] BLOCK if no-trade is omitted where the spec requires it.

## Encoding Integrity

- [ ] BLOCK if compared outcomes do not share scale.
- [ ] BLOCK if entry zones or stop pools lack named locations.
- [ ] BLOCK if forced exits or covering are not distinguished from fresh conviction.
- [ ] BLOCK if value migration is implied by color alone.
- [ ] BLOCK if hidden axes or inconsistent distances distort the read.

## Body Teaching

- [ ] BLOCK if the body cannot be read with rails covered.
- [ ] BLOCK if required evidence lives only in rail text.
- [ ] BLOCK if the first-pullback or reentry test is visually weak.
- [ ] BLOCK if the decision consequence is detached from the deciding evidence.

## Density And Labels

- [ ] BLOCK if more than two callouts compete inside one outcome panel.
- [ ] BLOCK if price badges are not monospace.
- [ ] BLOCK if rail text clips or forces smaller type.
- [ ] STOP if a three-branch comparison cannot fit without reducing body evidence.

## Approval Boundary

- [ ] BLOCK if any artifact claims approval.
- [ ] BLOCK if a renderer, render QA, export, injection, or PNG snapshot is run for Layer-2.
- [ ] BLOCK if `qa.visual_snapshot` is changed.
