# Price Path Level Interaction Layout Rules

This candidate grammar governs CH06 structural-location maps for `price_path_level_interaction`.

## Canonical Composition

1. Title band with concept claim.
2. Main price-path and level panel.
3. Secondary location map panel.
4. Trigger or activation panel.
5. Decision or professional alternative panel where required.
6. Defended branch panel or lane.
7. Evidence rail.
8. Forbidden rail.
9. Caption and footer.

The main price-path panel is the dominant teaching region. It must show the structural level and population location before any outcome.

## Named Regions

- `title-band`
- `main-price-level`
- `population-map`
- `risk-geometry`
- `activation-path`
- `decision-alternative`
- `defended-branch`
- `evidence-rail`
- `forbidden-rail`
- `caption-footer`

Not every spec uses every region, but no spec may invent regions outside this set without composition review.

## Main Price-Level Rules

- Time moves left to right.
- Price up is up.
- The watched level is a dashed or emphasized horizontal line with monospace price badge.
- Entrant zones are bounded bands with endpoints, not vague halos.
- Stop pools sit beyond or beneath the named structure, with stack marks.
- Normal pullback and sweep zones are bracketed between level and pool.
- Reentry or activation trigger is a distinct node on the path.

## Secondary Map Rules

Use the secondary panel for the spec's main location diagnosis:

- entrant concentration for breakout entrant map;
- pain threshold and invalidation geometry for late entrant map;
- latent pool versus activation for stop pool map;
- user stop overlap with crowd pool for user-stop map.

The secondary panel must share the same named levels as the main panel.

## Decision Region Rules

Decision panels are compact but visible. They may contain:

- wait for retest;
- wait for branch evidence;
- evidence exit before activation;
- reduce size for structural stop beyond sweep;
- no-trade.

Decision alternatives are not motivational copy. They are structural risk choices tied to location.

## Defended Branch Rules

Every CH06 target spec requires a defended or dormant branch:

- hold above the level;
- bid refresh;
- sells absorbed;
- value migrates toward the continuation target;
- stop pool remains latent where applicable.

This branch must be present in the body, not only in a rail row.

## Mark Vocabulary

- Watched level: dashed horizontal line and price badge.
- Entrant zone: translucent band, bracket endpoints, cohort label.
- Early/late distribution: stacked density ticks or histogram along the entry band.
- Pain threshold: bracket crossing back into the band.
- Stop pool: stacked stop markers, distinct from user stop pin.
- User stop: single labeled pin or marker overlapping the pool.
- Evidence invalidation: structural line beyond pool or sweep zone.
- Activation path: arrow through trigger and pool to finite target.
- Defended branch: alternate path with hold and value-migration block.
- Professional alternative: decision tokens, not chart marks.

## Label Lanes

- Level labels: left of line or right-edge price badge.
- Zone labels: inside band if short, otherwise above with leader.
- Stop pool labels: outside pool, never on top of stop markers.
- User stop label: anchored to the individual pin, visually separate from crowd pool.
- Decision labels: two-line maximum per decision token.
- Forbidden labels: rail only unless a body guard label is necessary.

## Latent Versus Activated Pool

The same stop pool may appear in two states:

- latent: muted stack markers, no forced-flow arrow, label says latent or not active;
- activated: rejected state, reentry trigger, forced path through the pool.

Do not draw a pool as active before failure and reentry evidence.

## Split Rule

Keep one plate if all required evidence can fit with clear body marks and rail verification.

Recommend split or human composition review if:

- the user stop, crowd pool, sweep zone, failure trigger, alternatives, and defended branch cannot all remain legible;
- more than 11 evidence ids need distinct body marks;
- a decision panel must become a rail-only paragraph;
- the defended branch becomes too small to prevent the every-breakout-is-trap misread.

## Stop Conditions

Stop if:

- no named level exists;
- no named population location exists;
- stop pool is shown as a signal by itself;
- user stop and crowd pool are confused;
- professional alternative is missing where required;
- the defended branch is omitted;
- labels overlap, stack, or shrink below token scale.
