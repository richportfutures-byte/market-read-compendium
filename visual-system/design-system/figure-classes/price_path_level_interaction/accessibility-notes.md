# Price Path Level Interaction Accessibility Notes

These notes are candidate design intent. They require verification on real exports later.

## Redundant Encoding

- Watched levels use dashed line, price badge, and label.
- Entry zones use bounded bands with endpoints.
- Stop pools use stacked markers, not color alone.
- User stop uses a single pin shape, distinct from pool stacks.
- Activated paths use arrow direction and trigger node.
- Defended branches use hold shape and value block.
- Professional alternatives use decision tokens with labels.

## Grayscale Survival

The figure must remain legible without color:

- latent pool is marker stack without activation arrow;
- activated pool has trigger node and forced path;
- user stop is a pin, not another stack marker;
- early versus late entrants differ by position and density;
- defended branch differs by path shape and value block.

## Contrast

Required evidence labels use high-contrast text or semantic evidence/state tokens against verified surfaces. Muted text is reserved for non-essential scaffolding.

## Motion And Static Equivalent

Most CH06 target specs are static. If an interactive variant is later created, the static export must still show all required evidence and decision alternatives.

## Alt Text Requirements

Published figures require authored alt text stating:

- the structural level;
- the entrant or trapped population location;
- the stop pool or invalidation geometry;
- the trigger or latent condition;
- the defended or dormant branch;
- the professional decision consequence;
- the forbidden misread prevented.
